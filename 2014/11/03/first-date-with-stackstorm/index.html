The ‘st2’ CLI provides access to the triggers, rules, actions, and action executions in the system. Any information you may need about any integrations or automations can be retrieved via the command line

Beginner’s Guide to the Action Library

The action library is essentially the muscles of the system. It shows all actions that have been installed from integration packs, as well as anything that has been added to the system ad-hoc. Just like the other areas of the system it is accessed through the ‘st2’ command

For instance, to see a list of currently registered actions, you run:

These are the core actions that come with a minimum install of the system. Let’s try out the local action. It executes an arbitrary shell command on the box where St2 is running.

The output shows a great deal of information about the action invocation but of particular interest to us is the ‘result’ field. It shows us information about the action on a per host basis (more on this later). For instance, since we executed the ‘date’ command on localhost, we see the output of ‘date’ in the “stdout” field.

This particular example isn’t all that interesting. Let’s try it on some remote hosts.  Before we can access the hosts we need to create the stanley user and copy his public key out to these hosts.  The public key can be found at ‘/home/stanley/st2_rsa.pub’. I generally use Puppet for this.

Now things start to get a little interesting. We see that we can not only execute the same command on multiple hosts but we can do it in parallel. Greatly reducing convergence time.
